Merge k Sorted Lists

Q

Merge k sorted linked lists and return it as one sorted list. Analyze and describe its complexity.

E:

my Solution (Stupid？)

算法流程

这题我参考了合并两个排序序列的方法，如下：

  • 找到队列中最小的节点
  • 奖节点加入排序好的链表，节点向前推进
  • 直到只能找到空的，说明全部找完

扩展到k个队列有两种方法

  1. 每次合并两个，一个接着一个的合并

  2. 将上面方法找到两个链表中最小的值，改为找到k个链表中最小的值

时间复杂度

上述的两种方法是时间复杂度是一样的

对于算法2，需要比较N次，每次比较k个节点，故算法复杂度为O(kN)

对于算法1，需要比较k-1次，每次比较$\frac{N}{k}*i+\frac{N}{k}$（假设是均分）算法复杂度为O(kN)

Smarter Solution

1. 堆排序

将k个链表建一个最小堆（优先队列），每个节点是链表的head（共有$lg(k)$层）

再每次取出最小堆的顶，对应链表向前推进一格，再将其加入堆中（非空）。

直至堆全部取完。

堆的操作（以最大堆为例）

维护堆的性质

假设堆的左右子树都符合堆的性质，那么其操作就是将该节点左右下沉

  • 找到左右子树最大的节点
  • 和本节点比较，如果本节点小于其，则交换它们
  • 直到无须交换

算法复杂度是O(lgn)

建堆

其核心方法就是自底向上维护最大堆的性质

  • 从底层的根节点开始维护堆的性质
  • 直至根节点

算法复杂度是O(n)

堆排序

堆排序就是不断取出根节点（将根节点和末尾节点交换），然后维护堆的性质

而因为要取出N个节点，所以其算法复杂度为O(n*lgn)

该算法的复杂度

该算法和堆排序不一样的是，该算法是用k个排序好的链表建堆的

即将比较时间变成了O(lg k)（维护堆的性质）

其算法复杂度为O(n*lgk)

2. 分治合并

将k个链表两两合并，这样只需要比较lg(k)次，每次是$2^i\frac{N}{k}$,i是合并的层数，这样算法复杂度O(lgk*N)
